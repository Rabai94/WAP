# RECOMMENDATION ENGINE

Status: Draft
Version: 1.0
Owner: WAP Team

---

# 1. Ce este Recommendation Engine

Recommendation Engine reprezintă componenta responsabilă pentru generarea recomandărilor inteligente în cadrul ecosistemului WAP.

Scopul său nu este să recomande cât mai multe lucruri.

Scopul său este să recomande doar acele soluții care pot aduce valoare reală participantului și ecosistemului.

Fiecare recomandare trebuie să fie utilă, relevantă, justificată și realizată la momentul potrivit.

Recommendation Engine trebuie să colaboreze permanent cu Decision Engine și cu celelalte componente ale lui Wapy pentru a identifica cele mai bune oportunități disponibile.
# 2. Principiul relevanței

Recommendation Engine nu trebuie să recomande informații inutile.

O recomandare trebuie făcută doar atunci când există o probabilitate ridicată ca aceasta să aducă valoare participantului.

Scopul nu este creșterea numărului de recomandări.

Scopul este creșterea calității recomandărilor.

O recomandare inutilă reduce încrederea în ecosistem.

O recomandare utilă crește încrederea și valoarea ecosistemului.
# 3. Principiul interesului utilizatorului

Recommendation Engine nu trebuie să recomande produse sau servicii doar pentru existența unui parteneriat sau a unui comision.

Interesul participantului trebuie să aibă întotdeauna prioritate.

Dacă există mai multe soluții potrivite, Recommendation Engine trebuie să le prioritizeze în funcție de valoarea pe care o aduc participantului și ecosistemului.

Nicio recomandare nu trebuie influențată exclusiv de interese comerciale.
# 4. Principiul recomandărilor transparente

Recommendation Engine trebuie să prezinte toate opțiunile relevante într-un mod clar, obiectiv și ușor de înțeles.

Atunci când există mai multe alternative, Wapy trebuie să explice pentru fiecare:

- ce competențe oferă;
- care dintre acestea sunt necesare pentru obiectivul actual;
- care reprezintă competențe suplimentare;
- avantajele și dezavantajele;
- costurile și timpul estimat;
- dacă există variante gratuite sau finanțate.

Wapy nu trebuie să favorizeze automat o opțiune gratuită sau o opțiune plătită.

Recomandarea trebuie să fie bazată pe valoarea pe care fiecare alternativă o aduce participantului în raport cu obiectivul său actual.

Atunci când mai multe opțiuni sunt potrivite, alegerea finală aparține participantului.

Rolul Wapy este să ofere informațiile necesare pentru luarea unei decizii informate, nu să decidă în locul utilizatorului.
# 5. Principiul momentului potrivit

Recommendation Engine nu trebuie să facă recomandări doar pentru că există o posibilă recomandare.

Momentul în care este oferită o recomandare este la fel de important ca recomandarea în sine.

O recomandare trebuie făcută atunci când participantul are cea mai mare probabilitate de a beneficia de aceasta.

Recommendation Engine trebuie să țină cont de contextul actual, obiectivele participantului, disponibilitatea acestuia și impactul recomandării.

Atunci când momentul nu este potrivit, Recommendation Engine poate decide să amâne recomandarea până când aceasta devine relevantă.

Scopul nu este maximizarea numărului de recomandări.

Scopul este maximizarea impactului fiecărei recomandări.
# 6. Principiul recomandărilor proactive

Recommendation Engine nu trebuie să aștepte întotdeauna ca participantul să solicite ajutor.

Atunci când identifică o oportunitate importantă sau un risc semnificativ, Wapy poate iniția o recomandare din proprie inițiativă.

Recomandările proactive trebuie utilizate cu responsabilitate și doar atunci când există o probabilitate ridicată ca acestea să aducă valoare participantului.

Exemple:

- apariția unui loc de muncă potrivit;
- lansarea unui curs relevant;
- expirarea unui program de finanțare;
- apariția unei oportunități de promovare;
- modificări importante pe piața muncii.

Recommendation Engine trebuie să evite recomandările excesive care pot reduce încrederea și atenția participanților.

# 7. Recomandări bazate pe potențial

WAP nu trebuie să recomande doar oportunitățile pentru care participantul este deja calificat.

Motorul de recomandări trebuie să identifice și oportunitățile pentru care participantul poate deveni eligibil prin dezvoltarea competențelor și calificărilor sale.

Recomandările trebuie împărțite în două categorii:

## Recomandări imediate

Oportunități pentru care participantul îndeplinește deja cerințele necesare și poate aplica imediat.

## Recomandări de dezvoltare

Oportunități pentru care participantul nu îndeplinește încă toate cerințele necesare.

În aceste situații, Wapy trebuie să explice clar participantului:

- ce competențe îi lipsesc;
- ce calificări sau certificări sunt necesare;
- ce cursuri sunt recomandate;
- dacă există programe gratuite sau finanțate;
- timpul estimat necesar pentru pregătire;
- costurile estimative, atunci când sunt disponibile;
- pașii recomandați pentru atingerea obiectivului profesional.

Participantul nu trebuie descurajat atunci când nu este eligibil pentru o oportunitate.

Scopul WAP este să transforme fiecare oportunitate într-un plan clar de dezvoltare profesională.

WAP nu trebuie să răspundă doar la întrebarea:

„Pentru ce pot aplica astăzi?”

WAP trebuie să răspundă și la întrebarea:

„Ce trebuie să fac pentru a putea aplica mâine?”
# 8. Diversitatea recomandărilor

Recommendation Engine nu trebuie să recomande permanent același tip de oportunități.

Recomandările trebuie să ofere participantului o perspectivă completă asupra posibilităților de dezvoltare.

În funcție de profilul și obiectivele participantului, Wapy poate recomanda:

- locuri de muncă;
- oportunități de promovare;
- cursuri și calificări;
- certificări;
- proiecte finanțate;
- programe gratuite;
- mentori;
- evenimente profesionale;
- companii relevante;
- oportunități antreprenoriale;
- alte recomandări utile.

Scopul Recommendation Engine este să descopere cea mai bună oportunitate pentru participant, nu doar cea mai apropiată.
# 9. Echilibrul recomandărilor

Recommendation Engine trebuie să păstreze un echilibru între recomandările pe termen scurt și cele pe termen lung.

Participantul trebuie să primească atât oportunități pe care le poate valorifica imediat, cât și recomandări care îl ajută să își atingă obiectivele viitoare.

Exemple:

- un loc de muncă disponibil astăzi;
- un curs care poate crește salariul în viitor;
- o certificare necesară pentru promovare;
- un program de recalificare;
- o oportunitate antreprenorială.

Scopul este ca Recommendation Engine să sprijine atât nevoile imediate, cât și dezvoltarea profesională pe termen lung.
